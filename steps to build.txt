A] omnetpp 4.4.1
1. Download zip 
2. download required packages
3. . setenv
4. set path of bin dir
5. ./configure
6. export TCL directory (command generated by configure last line)
7. ./configure (again)
8. make

B] Sumo should be 0.19.0
1. Download sumo-0.19.0
2. download required packages (use libproj-dev for proj)
3. ./configure
4. make
5. sudo make install


C] INETMANET
1. Download from https://github.com/jlsirois/
2. Delete undertest from src
3. Delete 8021_d from examples
4. Make makefiles
5. Make

	Optionally
	1. Import to eclipse
	2. Build

D] VANETProject
1. Download from https://github.com/the-elves/ITS
2. !!DO NOT BUILD FROM ECLIPSE
3. on commandline go to VANETProject/VANETSim
4. path in makemakefiles should point to inetmanet assuming strating 	from inside src.
5. make -f makemakefiles
6. make
7. run VANETSim folder from ecliplse
  -- error log
	1. if define_module problem, try building with opp in run configurations
	2. launch config is to be given in xmldoc("****") taken from omnetpp.ini of simulations
	3. add base dir tag eg:<basedir path="/home/administrator/ITS/VANETProject/VANETSim/simulations/_maps/simple/"/> (should point to directory containing map files)
	4. remove the path if exists from sumo.cfg files 
	5. set default for module_type parameter in TraciScenarioManager.ned to the value found in omnetpp.ini of aodvvanet 
	6. set the default for networkconfiguratormodule to "" in inetmanet/src/networklayer/autorouting/ipv4/ipv4NodeConfigurator
	7. rsu mobility parameter should be taken from omnetpp.ini of aodvvanet
